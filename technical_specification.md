# Техническое задание: Приложение для сканирования банковских платежных поручений

## 1. Общее описание
Разработка Android-приложения, позволяющего пользователю с помощью камеры телефона сканировать банковские платежные поручения (БП), распознавать реквизиты на русском языке (кириллица) и сохранять их в локальной базе данных с возможностью последующей выгрузки.

## 2. Цели
- Автоматизация ввода информации из бумажных платежных поручений.
- Снижение ошибок при ручном вводе.
- Упрощение процесса подготовки данных для бухгалтерии или финансового отдела.

## 3. Основные функции
### 3.1 Сканирование документа
- Использовать камеру устройства для захвата изображения платежного поручения.
- Поддержка как статических изображений, так и видеопотока с подсветкой границ документа (автоопределение контура).
- Возможность корректировки кадра (обрезка, поворот).

### 3.2 Распознавание текста
- Применить OCR для извлечения текста с изображения.
- Поддержка русского языка (кириллицы).
- Извлечение ключевых реквизитов:
  - Номер платежного поручения
  - Дата
  - Сумма
  - Получатель (ФИО/наименование организации)
  - Банк получателя
  - Реквизиты счета получателя (ИНН, КПП, номер счета)
  - Назначение платежа

### 3.3 Проверка и редактирование
- Отображение распознанного текста в виде таблицы/формы.
- Возможность ручной корректировки отдельных полей.
- Подсветка неуверенных участков для проверки.

### 3.4 Хранение данных
- Локальное хранение распознанных данных в SQLite (через Room).
- Структура записи:
  - id (INTEGER PRIMARY KEY)
  - document_image_path (TEXT)
  - recognition_timestamp (TEXT)
  - fields (JSON: { "number": "...", "date": "...", ... })
  - status (TEXT: "verified" | "pending")

### 3.5 Выгрузка данных
- Экспорт в CSV/Excel.
- Отправка по электронной почте или через API (если подключена серверная часть).
- Синхронизация с облачным хранилищем (опционально).

## 4. Технические требования
### 4.1 Платформа
- Минимальная версия Android: API 26 (Android 8.0)
- Рекомендуется использовать Kotlin + Jetpack Compose

### 4.2 Компоненты
- **Камера**: CameraX API
- **OCR**: ML Kit Text Recognition (поддержка кириллицы) или Tesseract OCR с русским языком
- **База данных**: Room Database
- **UI/UX**: Jetpack Compose, Material Design 3

### 4.3 Производительность
- Время распознавания ≤ 5 секунд на среднем устройстве.
- Поддержка изображений до 12 Мп.

## 5. Пользовательский сценарий
1. Пользователь открывает приложение.
2. Нажимает кнопку «Сканировать» → запускается камера.
3. Наводит камеру на платежное поручение → приложение выделяет границы документа.
4. Делает снимок → изображение обрабатывается (обрезка, улучшение контраста).
5. Запускается OCR → отображается распознанный текст в виде формы.
6. Пользователь проверяет/редактирует поля и сохраняет запись.
7. При нажатии «Выгрузить» данные экспортируются или отправляются.

## 6. Дальнейшие шаги
- Интеграция с облачным API (для централизованного хранения).
- Поддержка других типов документов.
- Обучение кастомной модели OCR для повышения точности на русскоязычных документах.

## 7. Примечания
- Приложение не должно требовать root-доступа.
- Все данные должны храниться локально по умолчанию (без отправки в облако без согласия пользователя).
